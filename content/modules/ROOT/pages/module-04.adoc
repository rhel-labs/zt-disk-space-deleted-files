= Confirming with lsof

== The Power of lsof

You've already discovered the deleted file by examining `/proc/$PID/fd/`. Now we're going to use one of the most powerful diagnostic tools in a system administrator's toolkit: `lsof` (List Open Files).

The `lsof` command shows all open files in the system, along with which process has them open. It's the standard tool for finding deleted files that are still consuming space, and it provides more detailed information than manually checking `/proc`.

== Finding Deleted Files

Let's search for any deleted files that are still open:

[source,bash,role=execute]
----
sudo lsof | grep deleted
----

[NOTE]
====
This command:
- Lists all open files in the system (`lsof`)
- Filters for lines containing "deleted" (`grep deleted`)

The output shows files that have been unlinked from the filesystem but are still open by running processes.
====

== Understanding the Output

The `lsof` output is formatted in columns:

- **COMMAND**: The name of the process
- **PID**: Process ID
- **USER**: User running the process
- **FD**: File descriptor (usually a number followed by a letter)
- **TYPE**: Type of file (REG = regular file)
- **DEVICE**: Device numbers
- **SIZE/OFF**: Size or offset (this shows how big the file is!)
- **NODE**: Inode number
- **NAME**: The filename, with "(deleted)" appended

[TIP]
====
Pay special attention to the SIZE/OFF column! This tells you exactly how much disk space the deleted file is consuming.

If you see a large number here (millions or billions of bytes), you've found your disk space thief!
====

== A More Targeted Search

We can be more specific and look just for deleted files in the `/var/log` area:

[source,bash,role=execute]
----
sudo lsof +L1 /var/log
----

[NOTE]
====
The `+L1` flag specifically shows files with a link count less than 1 - in other words, deleted files.

This is often cleaner than using `grep deleted` because it doesn't show other noise in the output.
====

== Getting Process Details

Now that we know which process has the deleted file open, let's get more information about it:

[source,bash,role=execute]
----
BUSINESS_PID=$(pgrep -f business-monitor.sh)
ps -p $BUSINESS_PID -f
----

This shows the full command line and details of the process.
